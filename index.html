🧠 Decisiones importantes para un ex usuario de Windows
No todo se instala con un .exe

En Linux se prioriza APT, repositorios, Flatpak, AppImage y paquetes .deb. Eso reduce descargas raras y actualiza desde un mismo sistema.

No edites producción directo

Tu PC local es laboratorio. GitHub guarda versiones. El servidor público solo debe recibir cambios probados.

Apache no es internet

Apache local solo responde en tu máquina o red. Para internet necesitas IP pública, DNS, puertos abiertos y HTTPS real.

Dominio local no se compra

{{LOCAL_DOMAIN}} vive en /etc/hosts. Un dominio real se compra y se apunta con DNS a una IP pública.

Servidor propio trae responsabilidad

Si no contratas hosting, tú cuidas parches, firewall, backups, certificados, monitoreo y energía.

La terminal no es castigo

Es una forma de repetir pasos exactos. Si un comando funciona, lo documentas y luego lo copias sin adivinar.

📦 Importar/exportar bases de datos
⚠️ Antes de importar sobre una base existente, haz respaldo. Importar un .sql puede crear tablas, insertar datos, modificar estructuras o borrar/reemplazar información si el archivo trae DROP TABLE, TRUNCATE, DELETE o CREATE OR REPLACE.
🧭 Idea simple

Exportar crea un archivo .sql desde una base. Importar toma un .sql y lo ejecuta dentro de una base. La flecha > escribe archivo; la flecha < lee archivo.

🧠 Regla práctica

Para importar, primero debes saber: qué archivo SQL usar, a qué base entrará, con qué usuario y si esa base ya existe.

🛡️ Seguridad

No pegues contraseñas en comandos. Usa -p sin contraseña pegada para que MariaDB la pida de forma oculta.

🧠 Qué significa importar/exportar realmente
Importar archivo .sql

Ejecuta dentro de MariaDB las instrucciones que vienen en el archivo. Puede crear tablas, insertar datos o cambiar estructuras.

Exportar base completa

Crea un respaldo con estructura y datos. Es lo normal antes de tocar algo importante.

Estructura vs datos

Estructura son tablas, columnas, índices y llaves. Datos son las filas guardadas dentro de esas tablas.

Base existente

Si importas encima de una base con datos, el resultado depende del contenido del .sql: puede mezclar, duplicar, fallar por llaves repetidas o reemplazar.

Archivo en Descargas

Puedes importar desde cualquier carpeta si escribes la ruta completa o si primero haces cd a esa carpeta.

DB origen y destino

Para copiar todo de una base a otra se exporta la base origen y se importa en la base destino, o se conecta con una tubería.
